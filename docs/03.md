### 风机排布优化是本项目最核心的计算模块，其设计质量直接影响选址结果的工程可用性。

#### 4.5.6 风机排布优化核心算法

【尾流模型设计】

- Jensen 尾流模型：采用线性尾流扩展假设，计算效率高，适用于平坦地形和平缓丘陵场景。该模型便于在大规模候选方案筛选阶段快速评估 AEP。
- Gaussian 尾流模型：采用高斯分布描述尾流扩散，精度更高，适用于机位更密集、地形更复杂或尾流干扰更显著的场景。该模型更适合作为精细优化与结果复核模型。
- 模型选择策略：在前期快速筛选阶段优先使用 Jensen 模型，在精细优化和结果校核阶段切换为 Gaussian 模型；对于复杂风况、复杂地形或多机型场景，可支持用户手动指定模型。

【机位排布优化建模】

优化问题可抽象为带约束的非凸连续优化问题：

- 决策变量：机位二维坐标 $(x_i, y_i)$，必要时可扩展为轮毂高度 $h_i$、偏航角 $\gamma_i$ 和机组运行状态。
- 约束条件：机位最小间距（通常不小于 $3D$）、场址边界约束、不可布机区域约束、道路/水系/坡度约束、施工红线约束等。
- 目标函数：最大化全场年发电量（AEP）或最小化平准化度电成本（LCOE）。在工程实现中，常将 LCOE 拆分为发电收益、集电成本、升压站成本和尾流损失成本等组成部分。
- 求解器选择：优先采用遗传算法（GA）或随机搜索增强的进化算法作为全局搜索主框架；对局部连续子问题，可叠加 SLSQP、IPOPT 等梯度型求解器做局部精修。

【推荐求解流程】

1. 生成候选机位集合，并剔除限制性区域内的点位。
2. 基于风资源与尾流模型计算初始 AEP / LCOE。
3. 采用 GA、PSO 或随机扰动搜索生成新布局。
4. 通过最小间距、边界、坡度等廉价约束进行快速过滤。
5. 对可行解调用 FLORIS 尾流模型进行目标函数评估。
6. 迭代更新至满足收敛判据或达到最大迭代次数。

【收敛与稳定性控制】

- 迭代代数建议不低于 500 代，复杂场景可增至 1000 代。
- 收敛判据建议采用连续多代最优值改善低于阈值的方式，例如 50 代内最优值变化小于 0.1%。
- 对随机算法建议设置多次独立运行，并对结果做统计稳健性比较，避免单次运行陷入局部最优。

#### 4.5.6.1 机型比选与异构机组布局

- 基于风资源特性的初筛：根据场址年均风速、湍流强度、剪切指数和风区等级，筛选满足 IEC 适配条件的机型集合。
- 精细比选：对候选机型分别运行布局优化，比较各方案的 AEP、LCOE、投资回收期和装机利用小时数。
- 综合评价：采用 TOPSIS、层次分析法（AHP）或加权评分法进行多指标排序，输出推荐机型或机型组合。

需要说明的是，FLORIS 对“异构机型”在建模层面是支持的：`turbine_type` 可以以列表形式输入，单个风场中可同时存在不同机型定义。但其内置布局优化器本身主要优化机位坐标和运行参数，并不直接把“机型选择”作为离散决策变量进行联合搜索。因此，如果要实现“集成风机类型的布局优化”，推荐采用以下两种路径：

- 路径 A：先做机型筛选，再对每个候选机型或机型组合单独运行布局优化，最后进行方案比较。
- 路径 B：把机型选择作为离散变量，引入混合整数优化或外层进化搜索，内层仍调用 FLORIS 评估 AEP / LCOE。

前者实现成本低、工程可控性强；后者更接近联合最优，但计算量明显更高。

#### 4.5.6.2 升压站选址优化

- 重心法：以各机位坐标为输入，按集电线路长度或机组容量加权，得到升压站候选中心位置，可作为初始解。
- 多因素加权评估：在重心法基础上叠加交通可达性、土地性质、坡度、地灾风险、用地红线和接入条件等因素，形成综合评分。
- 组合优化：将升压站选址与集电线路分配视为联合优化问题，目标为最小化“线路建设成本 + 升压站建设成本 + 损耗成本”。

#### 4.5.6.3 reV supply curve 逻辑对升压站选址的可用性

reV 的 supply curve 主要解决的是“供给点到电网接入点”的最优连接排序问题，其核心是按连接成本、可用容量和优先级进行筛选。其逻辑可以用于升压站选址优化中的“候选站点评估”环节，但不能直接替代完整的站址搜索问题。

- 可借用的部分：

- 以候选升压站为 POI（point of interconnection）或传输特征节点，计算机组到候选点的连接成本。
- 使用类似 `simple_sort` / `full_sort` 的排序机制，对候选站点进行成本优先级排序。
- 将线路成本、接入成本、扩容成本和容量约束统一到同一成本函数中。

- 不能直接复用的部分：

- supply curve 的原生场景是“已有网格特征的排序与连接”，不是连续或离散空间上的站址搜索。
- 它不负责生成站点候选集，也不负责处理复杂的空间布局约束和多目标平衡。

因此，升压站选址建议采用“候选点生成 + supply curve 风格成本评估 + 外层优化”的三段式方案：

1. 生成候选升压站集合，例如地块边界内的可建设点、道路附近可达点、已知规划站点等。
2. 对每个候选点计算集电线路长度、接入成本、容量利用率和扩建惩罚。
3. 通过 GA、MILP、模拟退火或 p-median 类位置分配算法，求解最优站址及机组分配方案。

#### 4.5.6.4 多场址优化的算法设计

对于多个风场同时优化的场景，问题通常会从单场址布局优化扩展为“多区域、多容量、多接入点”的联合优化，关键难点在于：不同风场之间可能共享升压站、送出通道或电网容量，且各风场的局部最优并不等于全局最优。

推荐采用以下几类算法：

- 分解协调算法：先按区域或并网点将问题分解，再通过协调变量统一各子问题的边界条件，适合大规模场景。
- 协同进化算法：每个风场作为一个子群体独立搜索，再通过共享信息协同进化，适合空间相互影响较弱但资源竞争明显的场景。
- 两阶段优化：第一阶段做风场级容量和站点预分配，第二阶段做单风场内部布局精修，便于工程落地。
- MILP / MINLP 联合优化：当候选点有限且约束明确时，可将选址、接入、容量分配建模为混合整数规划，适合中小规模问题。

工程上建议采用“主问题 + 子问题”的分层结构：主问题负责多个风场的站址、容量和通道分配，子问题负责每个风场内部的机位排布与尾流优化。这样既能保留全局协调能力，又能控制计算量。

#### 4.5.6.5 计算性能优化

- 并行计算策略：按风场、候选站点或风向情景拆分任务，各子任务独立运行并行求解，最终合并评价结果。
- 空间索引优化：使用 R-tree、Geohash 或网格哈希缓存候选机位的空间关系，加速间距约束、邻接约束和线路邻接检查。
- 预计算与缓存：风资源数据采用空间插值预生成高分辨率网格，布局迭代时直接查表，减少重复插值计算。
- 约束前置筛选：对明显越界或不可布机的方案提前过滤，避免把计算资源浪费在昂贵的尾流评估上。

### 4.6.5 多场址风机排布优化计算

多场址优化不是简单地把单场址求解器重复运行多次，而是需要显式考虑“多风场之间的资源竞争与网络耦合”。在本项目中，建议将多场址优化定义为一个层级化优化问题：

- 上层：决定各风场的开发优先级、共用升压站位置、送出通道和容量分配。
- 下层：在上层给定的边界条件下，分别对各风场运行布局优化和尾流评估。

#### 推荐实现思路

1. 按区域或接入点将多个风场划分为若干簇，形成可并行求解的子问题。
2. 在每个子问题内使用 FLORIS 或 reV 风格的成本函数进行局部优化。
3. 在全局层引入共享资源约束，例如升压站容量、送出线路容量、道路通行能力和土地指标。
4. 通过外层协调器统一更新子问题的惩罚项或拉格朗日乘子，逐步收敛到全局可行解。

#### 适用算法

- 协同进化 GA：适合风场之间耦合较弱、但候选方案数量庞大的情况。
- Benders 分解：适合“站址 / 通道 / 容量”与“风场内部布局”可拆分的问题。
- ADMM：适合需要在多个风场之间进行容量协调与并行求解的场景。
- 图优化 / 最小费用流：适合风场与升压站、集电线路之间的连接结构较清晰、候选节点有限的场景。

#### 结论性说明

- FLORIS 更适合作为“单风场布局与尾流评估引擎”。
- reV 更适合作为“供给点到接入点的成本与排序引擎”。
- 多风场联合优化应当放在系统层，由外层优化器统筹风场、升压站和送出网络，内部再调用 FLORIS / reV 做局部评估。

#### 方案建议

- 若目标是快速落地，建议先实现“候选点筛选 + 单风场布局优化 + 升压站成本排序”的组合方案。
- 若目标是全局最优，建议后续引入“分层优化 + 协同进化 / 分解协调”的多风场联合求解框架。
