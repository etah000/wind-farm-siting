# 风电场布局优化技术方案调研文档（FLORIS vs PyWake+TopFarm 及其他开源方案）

## 1. 调研概述

本文档针对**风电场智能化布局优化**场景，对比主流开源工具：NREL **FLORIS**、DTU **PyWake + TopFarm**（含 OptiWindNet 扩展），并补充 NREL **Ard** 等其他方案。重点围绕以下核心需求展开系统对比：

- 指定区域内风机排布优化（类似 FLORIS 的尾流建模 + 布局优化）
- 单风场统一风机类型下，**风机机型、风机台数、布局坐标**的联合优化
- 输电接入点（升压站位置、电缆路由）优化
- 多风电场分布式并行优化 + 结果对比

基于最新开源仓库（2026 年数据）和文献验证，输出工程落地推荐方案。

## 2. 核心业务需求定义

- **基础布局优化**：自定义边界、最小间距、复杂地形约束 + 主流尾流模型。
- **多参数联合优化**：统一风机类型前提下，同步优化机型、台数、坐标。
- **电气配套优化**：输电接入点、升压站、电缆路由 + 气动-电气一体化。
- **多风场分布式优化**：并行多个独立风场，自动汇总 AEP/LCOE 等指标对比。
- **高效算力**：支持智能优化（非纯网格穷举）、多核/集群分布式计算。

## 3. 主流开源方案介绍

### 3.1 FLORIS（NREL）
- **定位**：高精度尾流仿真 + 布局优化工具（Python，Apache 2.0）。
- **优势**：尾流模型丰富（7+ 种），工业验证充分，易用性高。
- **GitHub**：https://github.com/NREL/floris

### 3.2 PyWake + TopFarm（DTU）
- **PyWake**：高性能尾流仿真引擎（对标 FLORIS）。
- **TopFarm**：基于 **OpenMDAO** 的专业风场优化框架，支持多参数、多学科优化（MIT/AGPL）。
- **GitLab**：DTUWindEnergy 下的 TOPFARM/PyWake 和 TopFarm2。
- **定位**：全栈优化，强在联合优化与分布式计算。

### 3.3 OptiWindNet（DTU 官方电气扩展）
- 专用于海上/陆上风场**电气网络优化**（电缆布局、升压站选址等）。
- **GitHub**：DTUWindEnergy/OptiWindNet。
- 可与 PyWake + TopFarm 无缝集成，实现气电联合优化。

### 3.4 Ard（NREL / WISDEM 生态）
- 新兴模块化、多保真度、多学科优化套件。
- 支持梯度使能的布局优化，可扩展 BOS（含电气）成本模型。
- **GitHub**：NLRWindSystems/Ard（或 WISDEM 相关）。
- 适合与 FLORIS 结合使用。

## 4. 核心技术能力对比

### 4.1 基础布局优化能力
- **FLORIS**：优秀，支持复杂边界、梯度/随机/网格优化模式，精度顶尖。
- **PyWake + TopFarm**：对标精度，支持任意多边形边界、排除区、地形约束。

两者尾流仿真精度相当，TopFarm 更易集成高级约束。

### 4.2 机型、台数、布局联合优化（核心差异）
- **FLORIS**：原生仅支持**固定台数 + 固定机型**下的坐标优化。联合优化需**外层手动枚举**（台数区间 + 机型列表），内层 FLORIS 优化布局，属于二次开发实现，计算冗余大，易局部最优。
- **PyWake + TopFarm**：**原生支持**将台数（离散）、机型（离散）、坐标（连续）纳入统一优化框架。采用遗传算法（全局搜索）+ 梯度算法（局部求精，如 SLSQP）混合策略，支持**非网格穷举**的一体化优化。

**优势**：搜索效率和解质量显著更高。

### 4.3 输电接入点优化能力
- **FLORIS**：无内置电气模块，需完全自主二次开发耦合。
- **PyWake + TopFarm + OptiWindNet**：OptiWindNet 是 DTU 官方配套库，可无缝集成 PyWake（气动）与 TopFarm（优化），支持升压站选址、电缆拓扑、网损、建设成本等，实现**气电协同联合优化**。接口标准化，落地难度低。
- **Ard**：模块化设计，便于集成自定义 BOS/电气成本模型，潜力较大。

### 4.4 多风场分布式优化与对比
- **FLORIS**：无原生支持，需手动多进程/Dask 实现，结果汇总自研。
- **PyWake + TopFarm**：基于 **OpenMDAO + MPI** 原生支持分布式并行（单机多核 / 多节点集群 / SLURM），可并行多个风场任务，自动汇总指标对比。
- **Ard**：模块化，便于脚本化并行多个场景。

## 5. 优化算法与运行效率对比
- **FLORIS + 枚举**：冗余计算多，效率较低。
- **TopFarm**：混合算法 + OpenMDAO 高效求解器，整体效率显著优于枚举方案。
- 支持多保真度模型，进一步加速大批量优化。

## 6. 分布式计算能力
- **TopFarm**：MPI 原生支持，集群友好。
- **FLORIS**：需自定义实现。
- 所有方案均可结合 Dask / Ray 增强脚本化并行。

## 7. 方案综合对比总表

| 对比维度              | FLORIS                  | PyWake + TopFarm (+ OptiWindNet) | Ard (补充)          |
|-----------------------|-------------------------|----------------------------------|---------------------|
| 基础布局优化          | 优秀                    | 优秀（对标）                     | 优秀（模块化）      |
| 机型/台数/布局联合    | 需手动枚举              | **原生一体化**                   | 强（多学科）        |
| 输电接入点优化        | 无，需自研              | **官方 OptiWindNet 集成**        | 可扩展 BOS          |
| 多风场分布式          | 需自研                  | **原生 MPI**                     | 脚本化支持          |
| 优化算法              | 固定参数 + 枚举         | 遗传 + 梯度混合                  | 梯度使能多保真      |
| 计算效率（多参数）    | 中低                    | 高                               | 高                  |
| 开发/落地成本         | 高                      | 低                               | 中                  |

## 8. 方案选型结论与建议

**推荐优先级**：
1. **PyWake + TopFarm + OptiWindNet**（**最优全栈方案**）：全面覆盖所有核心需求，原生联合优化 + 电气集成 + 分布式能力最强，工程落地性最佳。
2. **Ard + FLORIS**：适合需要更强多学科/多保真度优化的场景，作为 TopFarm 的补充。
3. **纯 FLORIS**：仅需简单布局仿真或已有成熟 FLORIS 生态时使用。

**使用建议**：
- 快速验证：从 TopFarm2 示例开始（GitLab）。
- 电气重点：优先集成 OptiWindNet。
- 大规模多风场：利用 MPI/SLURM 集群。
- 混合方案：TopFarm 做联合优化，FLORIS 做高精度验证。

## 9. 总结与扩展建议

FLORIS 是优秀的**仿真引擎**，但在**多参数联合优化、电气集成、分布式**上存在明显短板。**DTU PyWake + TopFarm + OptiWindNet** 是当前开源领域最接近“全功能一体化”需求的方案。

**进一步扩展方向**：
- 集成机器学习代理模型加速优化。
- 支持混合风机类型（非统一类型）——TopFarm 有相关示例。
- 与 GIS/风资源数据（如 WAsP）深度结合。
- 监控最新仓库更新（DTU GitLab 活跃）。
